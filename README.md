# ROBOTIS_OMX_SW

ROBOTIS OMX 로봇팔과 로봇 소프트웨어 개발 과정에서 사용한 핵심 개념을 정리한 저장소입니다.
ROS 2와 Docker를 중심으로 로봇 프로그램의 통신 구조와 개발 환경 관리 방법을 정리했습니다.

## Contents

* [ROS 2](./ROS2.md)
* [Docker](./Docker.md)

## Overview

로봇 소프트웨어는 카메라, 센서, 모터, 제어 알고리즘처럼 여러 기능이 서로 연결되어 작동합니다.
이 저장소에서는 이러한 기능들을 효율적으로 연결하기 위한 ROS 2와, 개발 환경을 일정하게 유지하기 위한 Docker의 개념을 정리했습니다.

## ROS 2

ROS 2는 로봇 내부의 여러 프로그램을 서로 연결해 주는 통신 시스템입니다.
카메라, 모터, 센서, 자율주행 프로그램을 각각 독립적으로 작성한 뒤, 노드와 토픽을 이용해 서로 데이터를 주고받게 할 수 있습니다.

## Docker

Docker는 프로그램이 실행되는 운영체제, 라이브러리, 설정 환경을 하나로 묶어 관리하는 도구입니다.
ROS 2처럼 설치 과정이 복잡하고 버전에 민감한 개발 환경을 안정적으로 유지하는 데 사용됩니다.

## Why ROS 2 and Docker?

ROS 2는 로봇의 여러 기능을 나누어 개발하고 연결할 수 있게 해 줍니다.
Docker는 같은 개발 환경을 다른 PC나 로봇 하드웨어에서도 동일하게 실행할 수 있도록 도와줍니다.

따라서 ROS 2와 Docker를 함께 사용하면 로봇 소프트웨어를 더 체계적으로 개발하고, 환경 차이로 인한 오류를 줄일 수 있습니다.
