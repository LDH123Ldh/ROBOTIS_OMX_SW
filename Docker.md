# Docker

Docker는 프로그램이 실행되는 운영체제, 라이브러리, 설정 등을 하나로 묶어 관리하는 도구입니다.  
쉽게 말해 개발 환경 전체를 담아 두는 독립된 실행 공간이라고 볼 수 있습니다.

## 왜 Docker를 사용하는가?

로봇 소프트웨어를 개발할 때는 ROS 2 버전, Python 버전, 라이브러리 버전 등이 중요합니다.  
개발자의 PC에서는 잘 작동하던 프로그램이 실제 로봇에서는 실행되지 않는 경우가 생길 수 있습니다.

Docker를 사용하면 같은 개발 환경을 그대로 복사해서 실행할 수 있기 때문에  
“내 컴퓨터에서는 되는데 로봇에서는 안 된다”는 문제를 줄일 수 있습니다.

## 핵심 개념

### Image

이미지는 프로그램과 실행 환경을 담은 설계도입니다.  
필요한 운영체제, 라이브러리, 설정이 포함되어 있습니다.

### Container

컨테이너는 이미지를 실제로 실행한 독립된 공간입니다.  
컨테이너 안에서 작업하더라도 실제 PC 환경과 분리되어 있어 비교적 안전하게 개발할 수 있습니다.

### Volume

볼륨은 컨테이너와 실제 PC의 폴더를 연결하는 기능입니다.  
컨테이너가 종료되어도 코드나 데이터가 사라지지 않도록 도와줍니다.

## ROS 2와 함께 사용하는 이유

ROS 2는 설치 과정이 복잡하고 버전 차이에 민감할 수 있습니다.  
Docker를 사용하면 ROS 2가 설치된 환경을 미리 만들어 두고,  
다른 PC나 로봇에서도 같은 환경으로 실행할 수 있습니다.

## 정리

Docker는 로봇 소프트웨어 개발 환경을 일정하게 유지하기 위한 도구입니다.  
ROS 2와 함께 사용하면 설치 과정과 버전 충돌 문제를 줄이고, 협업과 배포를 더 쉽게 할 수 있습니다.
